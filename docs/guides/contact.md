## Reporting Guidelines

### Bug Reports

When reporting bugs, please include:

1. **Environment Information**
   - Operating system
   - Browser/app version
   - Device information (for mobile)

2. **Steps to Reproduce**
   - Detailed step-by-step instructions
   - Expected behavior
   - Actual behavior

3. **Additional Information**
   - Screenshots or screen recordings
   - Error messages or logs
   - Network conditions (if relevant)

### Feature Requests

For feature requests, please provide:

1. **Problem Description**
   - What problem does this solve?
   - Who would benefit from this feature?

2. **Proposed Solution**
   - Detailed description of the feature
   - How should it work?
   - Any design mockups or wireframes

3. **Alternatives Considered**
   - Other ways to solve the problem
   - Why is this the best approach?

### Security Reports

**Please DO NOT** report security vulnerabilities in public issues.

Instead:
1. Email: security@medico24.com
2. Include detailed information about the vulnerability
3. Provide steps to reproduce (if safe to do so)
4. Allow time for us to investigate and fix the issue
5. We will coordinate disclosure timeline with you

## Time Zones

Our development team is distributed across multiple time zones:

- **UTC**: Coordinated Universal Time (primary reference)
- **EST/EDT**: Eastern Time (UTC-5/UTC-4)
- **PST/PDT**: Pacific Time (UTC-8/UTC-7)

We aim to provide support coverage during business hours in all major time zones.
